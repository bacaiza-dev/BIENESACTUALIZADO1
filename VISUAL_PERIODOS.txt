COMPONENTES NUEVOS

┌─ Vue (Formulario)
│  ├─ Campo: input[type=number] para Año
│  │  └─ @change="updatePeriodoType" ← Detecta cambios
│  │
│  └─ Botón: button.btn-suggest
│     ├─ @click="autoSuggestDates" ← Función principal
│     ├─ :disabled="!form.tipo || !form.anio" ← Control de acceso
│     ├─ Icono: 🔄
│     └─ Mensaje: "✓ Fechas sugeridas automáticamente"
│
├─ Script (Lógica)
│  ├─ Variable: suggestedDates (ref(false))
│  │  └─ Indica si fueron sugeridas automáticamente
│  │
│  ├─ Función: autoSuggestDates()
│  │  ├─ Valida tipo y año
│  │  ├─ Calcula fechas según tipo
│  │  ├─ Detecta período actual
│  │  ├─ Actualiza form.fechaInicio y form.fechaFin
│  │  ├─ Muestra toast de éxito
│  │  └─ Establece suggestedDates = true
│  │
│  ├─ Función: updatePeriodoType()
│  │  ├─ Solo resetea suggestedDates a false
│  │  └─ No auto-genera fechas
│  │
│  └─ Función: closeModal()
│     └─ Resetea suggestedDates = false

VENTAJAS DE ESTA IMPLEMENTACIÓN

✅ Intuitivo      - El botón 🔄 es claro y fácil de entender
✅ No invasivo    - Solo sugiere, no obliga
✅ Inteligente    - Detecta el período actual automáticamente
✅ Flexible       - Permite edición manual de fechas
✅ Validado       - El botón solo se activa con datos válidos
✅ Visual         - Muestra confirmación de sugerencias
✅ Accesible      - Funciona en todos los navegadores
✅ Sin cambios de BD - Usa la estructura existente
✅ Rápido         - Cálculos instantáneos
✅ Documentado    - Incluye guías y tutoriales

PRUEBA RÁPIDA

1. Ve a: Gestión de Períodos Académicos
2. Haz clic: "Nuevo Período"
3. Llena:
   - Nombre: "Periodo Test"
   - Año: 2026
   - Tipo: Semestre
4. Haz clic: 🔄 (botón de actualizar)
5. Resultado esperado:
   - Fecha inicio: 01/01/2026 (si estamos en ene-jun)
   - Fecha fin: 30/06/2026 (si estamos en ene-jun)
   - ✓ Fechas sugeridas automáticamente (aparece)
6. Haz clic: "Crear"
7. ¡Listo! ✅
